## Header

Title: Seller views usage record detail and approves submissions
ID: US-012
Priority: Must
Actor: seller
As a: seller
I want: tap on a usage record name to view its detail page, optionally select a reference usage record and per-buyer replacement requests, see consumption, and approve or decline each buyer's submission
So that: I can verify each buyer's utility readings and calculated consumption before the data is used for billing

## Business Value

The usage record detail page is where the seller confirms electricity and water readings submitted by buyers before generating bills. The seller can optionally select a reference usage record to see how much each buyer has consumed since the previous period, and can assign a replacement request per buyer to account for meter swaps. Without this screen, sellers cannot review or approve submissions, blocking the entire billing workflow.

## Preconditions

- Seller is logged in
- At least one usage record exists for this seller

## Trigger

- Seller taps on the name of a usage record on the usage record list screen

## Acceptance Criteria

> Each criterion must be testable and written from the user's or system's observable perspective — no API details.

1. Tapping a usage record name navigates the seller to the usage record detail page.
2. The detail page displays the usage record name as the page title.
3. The page has a **Reference usage record** selector at the top. Seller can select any other usage record belonging to the same seller as the reference. This field is optional and defaults to empty.
4. When no reference usage record is selected, the table displays the following columns:
   - **Buyer name**: the buyer's `full_name`; vertically merged across the electricity and water rows of the same buyer.
   - **Items**: one row labeled "điện" (electricity) and one row labeled "nước" (water) per buyer.
   - **[Usage record name]**: column header is the name of this usage record; displays the submitted `electricity_current` on the electricity row and `water_current` on the water row.
5. When a reference usage record is selected, the table adds two additional columns:
   - **[Reference usage record name]**: displays the approved/manual value from the reference usage record for the corresponding buyer and item. If the buyer has no approved/manual value in the reference record, the cell is shown as empty.
   - **Lượng sử dụng**: calculated per item row as: current submission value − reference value. If the reference cell is empty, this cell is also empty.
6. Each buyer row has an optional **Replacement request** selector. Seller can assign one replacement request of the matching type (electricity or water) for that buyer. When a replacement request is assigned, the **Lượng sử dụng** for the affected item row is recalculated as: `(current submission value − new_reading) + (old_reading − reference value)`, where `old_reading` and `new_reading` come from the selected replacement request.
7. Only buyers who have submitted to this usage record are shown in the table.
8. All submissions are shown regardless of status (pending, approved, discarded, or manual).
9. If no submissions exist for this usage record, the table shows an empty state with a descriptive message.
10. Each buyer row has two actions available: **Approve** and **Decline**. These actions apply to the buyer's entire submission (both electricity and water readings together).
11. When the seller taps **Approve**, the submission status changes to `approved` and the table reflects the updated status immediately.
12. When the seller taps **Decline**, the submission status changes to `discarded` and the table reflects the updated status immediately.
