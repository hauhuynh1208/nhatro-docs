## Header

Title: Seller views usage record detail and approves submissions
ID: US-012
Priority: Must
Actor: seller
As a: seller
I want: tap on a usage record name to view its detail page with a consumption table and approve or decline each buyer's submission
So that: I can verify each buyer's utility readings against their baseline before the data is used for billing

## Business Value

The usage record detail page is where the seller confirms electricity and water readings submitted by buyers before generating bills. Without this screen, sellers cannot review or approve submissions, blocking the entire billing workflow.

## Preconditions

- Seller is logged in
- At least one usage record exists for this seller

## Trigger

- Seller taps on the name of a usage record on the usage record list screen

## Acceptance Criteria

> Each criterion must be testable and written from the user's or system's observable perspective — no API details.

1. Tapping a usage record name navigates the seller to the usage record detail page.
2. The detail page displays the usage record name as the page title.
3. The page displays a table with the following columns in order:
   - **Buyer name**: the buyer's `full_name`; vertically merged across the electricity and water rows of the same buyer.
   - **Items**: one row labeled "điện" (electricity) and one row labeled "nước" (water) per buyer.
   - **Baseline**: displays the buyer's `electricity_baseline` on the electricity row and `water_baseline` on the water row.
   - **[Usage record name]**: column header is the name of this usage record; displays the submitted `electricity_current` on the electricity row and `water_current` on the water row.
   - **Lượng sử dụng**: calculated as the usage record value minus the baseline value, shown per item row.
4. Only buyers who have submitted to this usage record are shown in the table.
5. All submissions are shown regardless of status (pending, approved, discarded, or manual).
6. If no submissions exist for this usage record, the table shows an empty state with a descriptive message.
7. Each buyer row has two actions available: **Approve** and **Decline**. These actions apply to the buyer's entire submission (both electricity and water readings together).
8. When the seller taps **Approve**, the submission status changes to `approved` and the table reflects the updated status immediately.
9. When the seller taps **Decline**, the submission status changes to `discarded` and the table reflects the updated status immediately.
