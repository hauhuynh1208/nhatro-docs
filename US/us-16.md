## Header

Title: Seller creates a sheet config
ID: US-016
Priority: Must
Actor: seller
As a: seller
I want: create a sheet config by defining columns with dummy data, assigning variables and formulas to them, and previewing calculated results
So that: I have a reusable billing template that can be applied when generating bills for my buyers

## Business Value

A sheet config is the billing template that defines which values are captured, how they are named, and how bill amounts are calculated. Without it, sellers cannot generate bills. The column-based configuration with dummy data and a live preview lets sellers verify their setup before applying it to real billing cycles.

## Preconditions

- Seller is logged in
- At least one variable exists for this seller (to assign to columns)

## Trigger

- Seller taps “Add Sheet Config” on the sheet config list screen

## Acceptance Criteria

> Each criterion must be testable and written from the user’s or system’s observable perspective — no API details.

1. Tapping “Add Sheet Config” opens a creation screen.
2. The screen includes a **Name** field (required, unique per seller).
3. The screen displays a single dummy buyer row with the following built-in columns. All values are dummy (for configuration purposes only) and default to:
   - **people_count** — default: 1 (editable)
   - **room_price** — default: 0 (editable)
   - **Old electricity reading** — default: 0 (editable, represents dummy old usage record — điện)
   - **Old water reading** — default: 0 (editable, represents dummy old usage record — nước)
   - **New electricity reading** — default: 0 (editable, represents dummy new usage record — điện)
   - **New water reading** — default: 0 (editable, represents dummy new usage record — nước)
   - **Electricity used** — auto-calculated as new electricity reading − old electricity reading; not editable
   - **Water used** — auto-calculated as new water reading − old water reading; not editable
4. Seller can change the default values of all editable built-in columns. The dummy row updates immediately.
5. For each editable built-in column, seller can assign a variable from this seller’s variables. The assigned variable will receive that column’s value during bill calculation.
6. Seller can add custom columns. For each custom column, seller assigns either:
   - A **variable**: the column holds a named value bound to that variable for use in formula calculations.
   - A **formula**: the column displays the result of the formula, computed using the current dummy values from all variable-assigned columns as inputs.
7. When a formula is assigned to a column, the dummy row shows a live preview of the formula result computed against the current dummy values.
8. Submitting with the Name field empty shows a validation error on that field.
9. Submitting with a Name already used by another sheet config of this seller shows a validation error.
10. On successful submission, the new sheet config appears at the top of the sheet config list.

## Notes

- The dummy data in the sheet config is only for configuration and preview. No real usage records are linked at this stage.
- When creating a bill, the seller selects which actual usage record is the old record and which is the new record. The sheet config template is then applied against those real values.
