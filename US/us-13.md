## Header

Title: Seller views replacement request list
ID: US-013
Priority: Must
Actor: seller
As a: seller
I want: view a list of all replacement requests I have created for my buyers
So that: I can track which meters have been swapped and their recorded readings

## Business Value

When a buyer's meter is physically replaced, the old and new readings must be recorded so that consumption can be calculated correctly across the meter change. A dedicated list screen gives sellers visibility into all past replacement events and serves as the entry point for managing them.

## Preconditions

- Seller is logged in

## Trigger

- Seller navigates to the Replacement Requests section from the main menu

## Acceptance Criteria

> Each criterion must be testable and written from the user's or system's observable perspective — no API details.

1. Seller can access the replacement request list screen from the main menu.
2. The list displays all replacement requests created by this seller, sorted by creation date with the most recent first.
3. Each row displays: buyer name, type (điện / nước), old reading (x), new reading (a), and creation date.
4. If no replacement requests exist, the screen shows an empty state with a descriptive message.

## UI/UX References

- None
